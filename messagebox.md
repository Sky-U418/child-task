# 消息框设计方案 v2

## 核心交互

- **面板内**：始终显示最新一条消息，不自动消失
- **历史查看**：点击消息框，弹出浮窗显示最近 24 小时内所有消息
- **新消息无消息时**：整行隐藏，不给空框占位

```
┌─────────────────────────────┐
│       可用积分: 156          │
│  🔥7天 1.7x │ 基础 │ 成果  │
├─────────────────────────────┤
│ 📡 太棒了！你刚完成了「…」  │  ← 始终显示最新一条
├─────────────────────────────┤
│  任务卡片 / 商店 / 我的      │
└─────────────────────────────┘

点击消息框 →
┌─────────────────────────────┐
│  消息记录（最近24小时） [×]  │
│                             │
│  2小时前                    │
│  🌟 完成了「整理书桌」+13分  │
│                             │
│  5小时前                    │
│  📡 新每日任务「浇花」+5分   │
│                             │
│  昨天 21:30                 │
│  🚀 连续7天打卡！倍率1.7x   │
└─────────────────────────────┘
```

## 位置

状态栏（积分横幅 + 三栏状态盒）与内容面板之间。

---

## 消息框样式

- 半透明深色背景 + 细边框
- 左侧类型颜色标识条（`border-left` 3px）
- 图标 + 文字，单行省略（`text-overflow: ellipsis` + `white-space: nowrap`）
- 右侧箭头 `▼` 暗示可点击展开
- `cursor: pointer`
- 有未读新消息时：微弱脉冲发光动画（`box-shadow` 亮度波动），点击查看后停止
- 入场动画：`opacity` + `translateY(-8px)`，200ms ease

---

## 浮窗样式

- 复用现有 `.report-detail-overlay` / `.report-detail` 结构
- 背景 `var(--color-bg-surface)`，圆角 `--radius-lg`
- 最大高度 60vh，超出滚动
- 消息按时间倒序排列
- 每条消息含：类型图标 + 文案 + 相对时间

## 消息类型与配色

| 类型 | 左边框色 | 图标 | 用途 |
|------|---------|------|------|
| 鼓励 `encourage` | 绿色 `--color-success` | 🌟 | 完成任务、达到里程碑 |
| 提醒 `remind` | 金色 `--color-warning` | ⏳ | 限时任务快到期、奖励即将过期 |
| 新增 `new` | 蓝色 `--color-accent` | 📡 | 管理员新增任务/奖励 |
| 系统 `system` | 紫色 `--color-info` | 🚀 | 打卡变化、倍率提升 |

## 数据结构

消息维护在 `child-app.js` 内存数组中（不入 Firestore，纯客户端逻辑）：

```js
// 消息对象
{
  id: 'msg_' + Date.now() + '_' + Math.random().toString(36).slice(2, 6),
  type: 'encourage',       // encourage | remind | new | system
  icon: '🌟',
  text: '太棒了！你刚刚完成了「整理书桌」+13分！',
  createdAt: Date.now(),   // 用于 24h 过滤和浮窗排序
  read: false              // 是否已点击查看
}
```

- 新消息 `unshift` 到头，面板取 `messageLog[0]`
- 浮窗展示 `messageLog.filter(m => Date.now() - m.createdAt < 86400000)`
- 数组上限 ~200 条，超出自动截尾防内存泄漏

## 建议的消息内容

### 鼓励类

| 触发条件 | 消息文案 |
|----------|----------|
| 完成任意任务后 | 太棒了！你刚刚完成了「{任务名}」+{积分}分！ |
| 单日完成 >= 3 个任务 | 你今天已经完成 {n} 个任务了，势不可挡！ |
| 连续打卡达到 7 天 | 连续 7 天打卡！你的坚持让积分倍率提升到 {倍率}x |
| 积分倍率达到最大 | 积分最大加速，保持这个状态吧！|


### 提醒类

| 触发条件 | 消息文案 |
|----------|----------|
| 限时任务剩余 <= 2 天 | 「{任务名}」还有 {n} 天到期，抓紧完成哦 |
| 限时任务剩余 <= 1 天 | 「{任务名}」明天就到期了！ |
| 奖励兑换次数即将用完 | 「{奖励名}」本月还能兑换 {n} 次 |
| 有已完成未领取的任务 | 你有 {n} 个任务已完成，别忘了领取积分！ |


### 新增类

| 触发条件 | 消息文案 |
|----------|----------|
| 管理员新增每日任务 | 指挥官发布了新每日任务：「{任务名}」(+{n}分) |
| 管理员新增限时任务 | 新限时任务：「{任务名}」(+{n}分)，{日期}到期 |
| 管理员新增奖励 | 商店上新：「{奖励名}」，{n}积分即可兑换 |

### 系统类

| 触发条件 | 消息文案 |
|----------|----------|
| 打卡重置（中断后重新开始） | 新的一天，新的开始！今天的倍率是 {倍率}x |
| 每日重置（0点后首次加载） | 每日任务已刷新，新的一天继续加油！ |
| 倍率提升 | 连续打卡 {n} 天，积分倍率升至 {倍率}x！ |

## 实现方式

消息来源全部由客户端自动生成：

- 在 `child-app.js` 中维护 `messageLog` 数组
- `onSnapshot` 检测变化时对比前后状态，自动生成消息推送
- 任务数增多 → 新任务消息
- 任务状态变为 completed → 鼓励消息
- 即将过期 → 提醒消息
- 倍率变化 → 系统消息

后续可扩展管理员手动推送（写入 Firestore `messages` 集合）。
